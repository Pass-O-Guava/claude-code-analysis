## 1. 产品概述

### 1.1 产品定位

**Scale（AI模型测评官）** 是一款面向 AI 从业者的大模型自动化测评工具。用户只需提供一篇模型发布的文章链接，系统即可自动完成从信息提取到测评报告生成的全流程。

**核心价值主张:**
> "让模型测评像阅读文章一样简单"

### 1.2 目标用户

| 用户类型 | 痛点 | 使用场景 |
|----------|------|----------|
| AI 研究员 | 手动测评耗时耗力 | 快速验证新模型能力 |
| 技术博主 | 缺乏专业测评环境 | 生成测评文章素材 |
| 企业决策者 | 选型困难 | 模型横向对比 |
| 开发者 | 本地资源有限 | 云端测评服务 |

### 1.3 产品边界

**我们做什么:**
- 自动化测评流程编排
- 基准测试执行
- 可视化报告生成
- 测评历史管理

**我们不做什么:**
- 模型训练/微调
- 自建测评数据集
- 模型托管服务

## 2. 功能需求

### 2.1 核心功能

#### FR-001: 文章解析
**优先级:** P0  
**描述:** 自动从公众号文章、技术博客中提取模型信息  
**输入:** 文章URL  
**输出:** 模型名称、版本、架构、宣称能力、模型权重链接

#### FR-002: 环境准备
**优先级:** P0  
**描述:** 自动准备 GPU 环境和模型文件  
**流程:**
1. 检测/申请 GPU 资源
2. 下载模型权重
3. 加载到显存
4. 健康检查

#### FR-003: 基准测试
**优先级:** P0  
**描述:** 执行标准化测评  
**支持数据集:**
- MMLU (知识理解)
- GSM8K (数学推理)
- HumanEval (代码生成)
- C-Eval (中文知识)
- CMMLU (中文综合)

#### FR-004: 报告生成
**优先级:** P0  
**描述:** 生成专业测评报告  
**内容:**
- 综合评分
- 雷达图 (多维度能力)
- 与同规模模型对比
- 优缺点分析

### 2.2 辅助功能

#### FR-005: 实时进度
**优先级:** P1  
**描述:** WebSocket 实时推送测评进度

#### FR-006: 任务管理
**优先级:** P1  
**描述:** 查看历史测评任务、重新运行

#### FR-007: 报告导出
**优先级:** P2  
**描述:** 支持 PDF、Markdown 格式导出

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 文章解析 | < 30s | 提取模型基本信息 |
| 环境准备 | < 5min | GPU就绪、模型加载完成 |
| 单数据集测评 | < 10min | 取决于数据集大小 |
| 完整测评 | < 30min | 所有基准测试 |
| API 响应 | < 200ms | 健康检查等简单接口 |

### 3.2 可用性要求

- 系统可用性: 99.9%
- 支持并发: 3 个测评任务同时运行
- 故障恢复: 自动重试机制

### 3.3 安全要求

- 用户输入 URL 需校验
- 测评环境隔离
- 模型文件访问控制

## 4. 用户体验

### 4.1 交互流程

```
[输入URL] → [选择测评模式] → [确认开始]
                 ↓
        [显示进度条 (4步骤)]
                 ↓
        [展示测评结果]
                 ↓
        [导出/分享报告]
```

### 4.2 界面设计

#### 首页
- 简洁的 URL 输入框
- 测评模式选择 (快速/完整)
- 最近测评历史

#### 进度页
- 4 步骤进度条
- 实时日志输出
- 预计剩余时间

#### 结果页
- 综合评分 (大数字)
- 雷达图 (6维度)
- 详细指标表格
- 对比分析

### 4.3 错误处理

| 错误场景 | 提示方式 | 恢复建议 |
|----------|----------|----------|
| URL 无法访问 | Toast 提示 | 检查链接有效性 |
| GPU 资源不足 | 弹窗提示 | 稍后再试或选择快速模式 |
| 模型加载失败 | 日志显示 | 查看详细错误 |
| 测评超时 | 状态更新 | 部分结果展示 |
