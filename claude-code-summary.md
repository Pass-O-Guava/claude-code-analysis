# Claude Code 泄露事件摘要报告

**报告日期**: 2026年4月1日

---

## 泄露事件概述

2026年3月31日，Anthropic 的 AI 编程助手 Claude Code 遭遇严重源代码泄露。安全研究员 Chaofan Shou 在 npm 包中发现，Anthropic 意外地将包含完整源代码的 `.map` sourcemap 文件发布到了 npm 仓库。Sourcemap 文件是构建工具生成的映射文件，用于将压缩后的代码映射回原始源码——**它本质上包含了完整的原始源代码**。这是一个极其低级的配置错误：开发团队忘记在 `.npmignore` 中排除 `*.map` 文件，也未在构建时禁用 sourcemap 生成。

## 代码规模

泄露的源代码规模令人震惊：

- **约 513,000 行 TypeScript 代码**
- **约 1,900 个源文件**
- **单个入口文件 `main.tsx` 达 785KB**
- **40+ 个工具实现**
- **100+ 个斜杠命令**

GitHub 用户 Kuberwastaken 对该代码进行了 Clean-room 重写，将其转换为 Rust 实现，产生约 23,847 行 Rust 代码和 53 个源文件。

## 重大发现

泄露代码揭示了多个 Anthropic 从未公开的内部系统和产品计划：

**内部模型代号系统**：
- "Tengu" = Haiku 模型
- "Capybara" = Sonnet 模型
- "Titan" = Opus 模型
- "Fennec" = Opus 早期代号

**即将发布的模型**：
- Capybara v2-fast (支持 1M 上下文窗口)
- Opus 4.7 / Sonnet 4.8

**讽刺性的安全疏忽**：代码中包含一个名为 "Undercover Mode" 的系统，专门设计用于防止 AI 泄露内部代号——然而整个源代码却因 `.map` 文件被公开。

## 隐藏特性

泄露代码包含至少 **44 个未发布功能**，其中最引人注目的是：

**KAIROS - 始终在线模式**：一个后台守护进程系统，使 Claude 能够主动观察、记录并自主采取行动。它维护每日追加日志，在 15 秒阻塞预算内自主决策，拥有 SendUserFile、PushNotification 等独占工具。

**BUDDY 电子宠物系统**：一个完整的 Tamagotchi 风格伴侣系统，包含 18 个物种、5 层稀有度、1% 闪光率、Mulberry32 PRNG 确定性生成、ASCII 精灵渲染和动画系统。这是 AI 工具中前所未有的游戏化设计。

**Dream System - 记忆整合引擎**：一个后台子代理系统，通过"三闸门触发"（24小时时间门、5会话门、锁门）定期压缩和整合长期记忆。

**ULTRAPLAN - 远程规划**：将复杂规划任务 offload 到云端 CCR 会话，使用 Opus 4.6 进行长达 30 分钟的深度思考。

## 技术亮点

泄露代码展示了 Claude Code 的复杂架构：

- **自定义 React 终端渲染器**：基于 Ink 的自定义 TUI 框架
- **多代理协调系统**：支持子代理、后台代理和 Swarm 模式
- **云端同步架构**：通过 Bridge 协议实现 WebSocket/SSE 传输
- **完善的工具生态系统**：40+ 个工具覆盖文件、搜索、Web、任务等全场景

这次泄露不仅暴露了配置管理失误，更意外揭示了 Anthropic 正在构建的下一代 AI 交互范式：从被动响应转向主动协助，从工具转向伙伴。
